# ATS Optimization Guidelines

## ATS Compatibility Target
**Target Match Rate:** 65-75% for top Silicon Valley companies

## File Format Requirements

### Primary Format
- **Microsoft Word (.docx)** - Primary submission format
- **Never submit PDF** unless explicitly requested
- **Plain text backup** - Copy-paste into Notepad to verify parsing

### Formatting Rules
- **Single-column layout** - No tables, text boxes, graphics, or two-column designs
- **Standard fonts only**: Calibri, Arial, Helvetica, Times New Roman
- **Font sizes**: 10.5-12pt for body text, 16-20pt for name
- **Simple bullet points**: Use (•) only, no fancy bullets or symbols
- **ASCII punctuation**: Avoid special characters that may not parse correctly

## Section Structure

### Required Section Headings (Use Exactly)
- **PROFESSIONAL SUMMARY** (not "About" or "Profile")
- **CORE SKILLS** (not "Technical Skills" or "Expertise")
- **PROFESSIONAL EXPERIENCE** (not "Work History" or "Employment")
- **EDUCATION**
- **CERTIFICATIONS** (if applicable)

### Section Order
1. Header (contact information)
2. Professional Summary
3. Core Skills
4. Professional Experience
5. Projects/Platforms (optional)
6. Education
7. Certifications

## Contact Information Rules
- **Must be in document body**, never in headers/footers
- **Plain text URLs** - No hyperlinks or "click here" text
- **Standard format**: Phone: +1234567890 | Email: name@email.com
- **LinkedIn/GitHub**: Include full URLs in plain text

## Keyword Optimization

### Keyword Density
- **Target range**: 1-3% of total document
- **Avoid keyword stuffing**: >4% density flags as spam
- **Natural integration**: Keywords must read naturally in context

### Keyword Placement Priority
1. **Professional Summary** (3-5 priority keywords)
2. **Core Skills section** (15-25 items)
3. **Experience bullets** (embedded naturally)
4. **Project descriptions** (supporting keywords)

### 2026 High-Priority Keywords
**Programming Languages:** Python, TypeScript, JavaScript, Go, Java, C#
**Cloud Infrastructure:** Kubernetes, Docker, AWS, Azure, Google Cloud Platform
**AI/ML:** TensorFlow, PyTorch, MLOps, LLMs, Machine Learning
**Development:** React, Next.js, Node.js, FastAPI, REST APIs
**Data:** PostgreSQL, MongoDB, Redis, Data Pipelines, ETL
**DevOps:** CI/CD, Jenkins, Terraform, Monitoring, Prometheus

## Experience Section Optimization

### Date Format
- **Consistent format**: "Jan 2023 – Present" or "January 2023 – Present"
- **No abbreviations**: Avoid "1/23" or "Jan '23"
- **Use en dash (–)**: Not hyphen (-) between dates

### Bullet Point Format
- **Start with action verbs**: Built, Implemented, Architected, Designed
- **Use X→Y→Z formula**: Accomplished [X] measured by [Y] using [Z]
- **Include metrics**: Specific numbers, percentages, scales
- **Embed keywords naturally**: Technical stack mentioned in context

### Action Verb Examples
**Strong:** Built, Implemented, Architected, Designed, Optimized, Engineered, Developed, Created, Established, Led
**Avoid:** Responsible for, Worked on, Helped with, Involved in, Participated in

## Skills Section Optimization

### Format
- **Comma-separated list**: Not categorized or bulleted
- **15-25 total items**: Within optimal range for ATS parsing
- **Strategic ordering**: Most important/relevant keywords first
- **No skill levels**: Don't include "Expert," "Advanced," etc.

### Grouping Strategy
1. **Core programming languages** (Python, TypeScript, JavaScript)
2. **Primary frameworks/libraries** (React, Next.js, FastAPI)
3. **Unique differentiators** (AI/ML technologies, specialized tools)
4. **Infrastructure/DevOps** (Docker, Kubernetes, AWS)
5. **Databases/Data** (PostgreSQL, Redis, Data Pipelines)

## Testing and Validation

### ATS Testing Tools
- **Jobscan.io**: Upload resume + job description, target 65-75% match
- **ResumeWorded**: Free ATS checker and optimization suggestions
- **TopResume**: Professional ATS analysis service

### Manual Testing
1. **Plain text test**: Copy-paste entire resume into Notepad
2. **Format preservation**: Ensure structure remains intact
3. **Keyword density check**: Count keywords vs total words
4. **Readability test**: 6-8 second scan for key information

### Common ATS Failures
- **Headers/footers**: Contact info gets lost
- **Tables and columns**: Content gets scrambled
- **Special characters**: Non-ASCII characters cause parsing errors
- **Images/graphics**: Completely ignored by ATS
- **Fancy formatting**: Text boxes, borders, shading cause issues

## Optimization Checklist

### Pre-Submission Checklist
- [ ] Saved as .docx format
- [ ] Single-column layout verified
- [ ] Contact info in body text
- [ ] Standard section headings used
- [ ] Consistent date formatting
- [ ] 15-25 keywords in Skills section
- [ ] X→Y→Z format for experience bullets
- [ ] Plain text URLs for portfolios
- [ ] ATS test score 65%+
- [ ] Plain text formatting test passed

### Quality Assurance
- [ ] No marketing buzzwords ("cutting-edge," "innovative")
- [ ] All metrics are accurate and verifiable
- [ ] Technical keywords match actual experience
- [ ] Reading flow is natural and professional
- [ ] Bullet points start with strong action verbs
